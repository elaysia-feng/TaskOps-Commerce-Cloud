# TaskOps Cloud 项目整体流程图（复习版）

## 1) 系统总览（服务关系 + MQ）

```mermaid
flowchart LR
    U[用户/前端] --> N[Nginx]
    N --> G[Gateway Service]

    G --> A[Auth Service]
    G --> T[Task Service]
    G --> O[Order Service]
    G --> P[Pay Service]
    G --> AD[Admin Service]

    A --> R[(Redis)]
    T --> R
    O --> R
    P --> R

    A --> DBA[(MySQL auth)]
    T --> DBT[(MySQL task)]
    O --> DBO[(MySQL order)]
    P --> DBP[(MySQL pay)]

    O --> MQ[(RabbitMQ: taskops.business.exchange)]
    MQ --> P
    MQ --> O
    MQ --> T

    O -.->|order.created| MQ
    P -.->|pay.success| MQ
```

## 2) 登录与鉴权主链路（先登录，再访问业务）

```mermaid
sequenceDiagram
    participant User as 用户
    participant GW as Gateway
    participant Auth as auth-service

    User->>GW: POST /api/auth/login (账号密码)
    GW->>Auth: 转发登录请求
    Auth-->>GW: 返回 token
    GW-->>User: 登录成功 + token
```

```mermaid
sequenceDiagram
    participant User as 用户
    participant GW as Gateway
    participant Biz as task/order/pay/admin

    User->>GW: 业务请求 + Authorization token
    GW->>GW: 校验token并解析用户信息
    GW->>Biz: 转发请求并透传用户上下文
    Biz-->>GW: 业务响应
    GW-->>User: 返回结果
```

## 3) 会员付费主链路（当前改造后）

```mermaid
sequenceDiagram
    participant User as 用户
    participant GW as Gateway
    participant Order as order-service
    participant MQ as RabbitMQ
    participant Pay as pay-service
    participant Task as task-service

    User->>GW: 创建会员订单
    GW->>Order: POST /api/orders
    Order->>Order: 落库 status=PENDING_PAY
    Order->>MQ: 发布 order.created

    MQ->>Pay: 消费 order.created
    Pay->>Pay: createPaymentIfAbsent()

    User->>GW: 支付回调(模拟)
    GW->>Pay: POST /api/pay/callback/mock
    Pay->>Pay: 支付单置 SUCCESS
    Pay->>Order: 标记订单已支付 markPaid
    Pay->>MQ: 发布 pay.success(userId, skuCode)

    MQ->>Task: 消费 pay.success
    Task->>Task: 升级会员等级(FREE/VIP/SVIP)

    User->>GW: 创建任务
    GW->>Task: POST /api/tasks
    Task->>Task: 按会员额度校验(20/50/100)
    Task-->>GW: 通过则创建，超限则拒绝
```

## 4) 关键事件与队列

```mermaid
flowchart TD
    EX[taskops.business.exchange direct]

    O1[order.created] --> EX
    EX --> Q1[order.created.pay.queue]
    Q1 --> C1[pay-service OrderCreatedConsumer]

    O2[pay.success] --> EX
    EX --> Q2[pay.success.order.queue]
    Q2 --> C2[order-service DirectListener]

    EX --> Q3[pay.success.task.queue]
    Q3 --> C3[task-service PaySuccessConsumer]
```

## 5) 当前项目的业务定位

- 任务系统是核心域：`task-service`
- 订单/支付是支撑域：用于购买会员权益
- MQ用于服务解耦和异步通知，不直接替代业务规则
