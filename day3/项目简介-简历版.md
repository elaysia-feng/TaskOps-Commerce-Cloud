# TaskOps Console 项目简介（简历可用）

## 项目名称
TaskOps Console（任务协同控制台）

## 项目简介（精简版）
基于 Spring Cloud 微服务架构实现的任务协同平台，包含网关鉴权、用户认证、任务管理与管理端聚合查询；前端使用 Vue3 构建控制台界面，实现登录鉴权、任务分页检索与任务创建等核心流程。

## 技术栈
Java 11、Spring Boot、Spring Cloud Gateway、OpenFeign、MyBatis-Plus、MySQL、Redis、Nacos、Vue3、Vite、Axios、Nginx、Docker Compose

## 简历描述（可直接粘贴）
- 参与设计并开发 TaskOps Cloud 微服务系统，落地 `gateway/auth/task/admin` 四个核心服务与公共模块，完成从鉴权到任务业务的完整闭环。
- 在网关层实现 JWT 统一鉴权与 RBAC 基础校验，解析 Token 后通过请求头透传用户身份，微服务侧通过 `UserContext + ThreadLocal` 获取当前用户上下文，降低重复鉴权成本。
- 基于 MyBatis-Plus 实现任务分页检索与条件查询（关键词、状态、优先级区间），通过 `Page(current,size)` 完成高效分页，支持前端分页交互。
- 使用 Redis ZSet 设计任务热榜能力，结合定时任务重建热点数据，提升热门任务查询性能。
- 独立搭建 Vue3 前端控制台，完成认证区与业务区分离布局、多步骤登录流程、路由守卫、Axios 统一请求拦截与 Token 自动携带。

## 个人职责（实习表达）
- 负责网关鉴权链路开发（Token 校验、白名单、角色限制、用户信息透传）。
- 负责任务模块核心接口开发（创建、分页搜索、详情、热榜）与前后端联调。
- 负责控制台前端从 0 到 1 搭建，完成登录、注册、任务列表、创建任务等页面。

## 项目亮点（面试可讲）
- 统一鉴权架构：将鉴权前置到 Gateway，业务服务专注领域逻辑。
- 微服务上下文传递：通过请求头 + ThreadLocal 兼顾性能与代码整洁。
- 可扩展分页方案：分页参数与筛选条件解耦，便于后续扩展排序、多条件筛选。
- 前后端协同：从接口设计、鉴权约定到页面交互完成端到端闭环交付。
