# Redis 三大缓存问题：常见方式与企业常用方案

## 1. 缓存穿透
- 常见解决方式：
  - 空值缓存（null 也缓存，短 TTL）
  - 参数校验（非法 key 直接拦截）
  - 布隆过滤器（先判断 key 是否可能存在）
- 企业常用方案：
  - `布隆过滤器 + 空值缓存` 组合

## 2. 缓存击穿
- 常见解决方式：
  - 互斥锁重建缓存（单线程回源）
  - 逻辑过期 + 异步重建
  - 热点 key 永不过期 + 主动更新
- 企业常用方案：
  - 中小并发：`互斥锁`
  - 高并发热点：`逻辑过期 + 异步重建`

## 3. 缓存雪崩
- 常见解决方式：
  - TTL 随机抖动（打散过期时间）
  - 多级缓存（本地缓存 + Redis）
  - 限流 / 降级 / 熔断
  - Redis 高可用（哨兵/集群）
- 企业常用方案：
  - `TTL随机 + 限流降级 + Redis高可用` 组合

## 一句话记忆
- 穿透：查不存在 -> 布隆/空值。
- 击穿：热点失效并发回源 -> 锁/逻辑过期。
- 雪崩：大量同时失效 -> TTL 打散 + 保护数据库。
