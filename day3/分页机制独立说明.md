# 分页机制独立说明（TaskOps）

你问的核心：

- 前端传 `current(page)` 和 `pageSize(size)`
- 后端为什么只查 `pageSize` 条？

答案：因为 MyBatis-Plus 分页插件会把这两个参数转换成 SQL 的 `LIMIT` 语句。

---

## 1. 你项目里的真实调用链

1) 前端请求（当前项目字段名是 `page`、`size`）

- 文件：`taskops-frontend/src/views/TaskListView.vue`
- 请求体示例：

```json
{
  "keyword": "redis",
  "status": "TODO",
  "page": 1,
  "size": 10
}
```

2) 后端接收 DTO

- 文件：`task-service/src/main/java/com/elias/task/dto/TaskQueryRequest.java`
- 字段：`page`、`size`

3) Service 构造分页对象

- 文件：`task-service/src/main/java/com/elias/task/service/impl/TaskAppServiceImpl.java`
- 代码关键点：

```java
Page<InternshipTask> page = new Page<>(Math.max(1, request.getPage()), Math.max(1, Math.min(50, request.getSize())));
return taskMapper.selectPage(page, wrapper);
```

4) MyBatis-Plus 分页拦截器生效

- 文件：`task-service/src/main/java/com/elias/task/config/MybatisPlusConfig.java`
- 注册了：`PaginationInnerInterceptor(DbType.MYSQL)`

---

## 2. 它到底怎么变成 SQL

分页对象 `Page(current, size)` 会转换成：

- `offset = (current - 1) * size`
- `limit = size`

SQL 形态近似：

```sql
SELECT ...
FROM internship_task
WHERE ...
ORDER BY updated_at DESC
LIMIT offset, size;
```

示例：

- `current = 1, size = 10` -> `LIMIT 0,10`（查第 1-10 条）
- `current = 2, size = 10` -> `LIMIT 10,10`（查第 11-20 条）
- `current = 3, size = 10` -> `LIMIT 20,10`（查第 21-30 条）

所以后端“知道查多少条”，就是看 `size`，并且你这里还限制最大 50 条，防止一次查太大。

---

## 3. 返回给前端的是什么

`selectPage` 返回 `IPage`，里面通常有：

- `records`：当前页数据（长度 <= size）
- `total`：总条数
- `current`：当前页
- `size`：每页大小

前端再算总页数：

```js
totalPage = Math.ceil(total / size)
```

---

## 4. 你提到的 current/pageSize 命名

现在你的后端字段是 `page/size`，不是 `current/pageSize`。

如果你前端要传 `current/pageSize`，有两种做法：

1) 前端改名后再发（推荐，改动最小）
2) 后端 DTO 新增 `current/pageSize` 并做映射到 `page/size`

当前项目不改也没问题，直接统一用 `page/size` 即可。

---

一句话结论：

`page(current)` 决定从哪一条开始，`size(pageSize)` 决定取几条；MyBatis-Plus 分页插件自动把它们翻译成 `LIMIT offset,size`。
