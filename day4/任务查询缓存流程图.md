# 任务查询流程图（含缓存与反序列化）

## 1) 任务热榜查询主流程

```mermaid
flowchart TD
    A[前端请求 /api/tasks/hot] --> B[TaskController.hot]
    B --> C[TaskAppServiceImpl.hot]
    C --> D[Redis ZSet reverseRange HOT_KEY 0~9]
    D --> E{ids是否为空}
    E -- 是 --> F[rebuildHot 重建热榜]
    F --> G[再次读取ZSet]
    G --> H{仍为空?}
    H -- 是 --> I[返回空列表]
    H -- 否 --> J[遍历ids]
    E -- 否 --> J
    J --> K[queryTaskWithCacheProtection(taskId)]
    K --> L[返回任务列表]
```

## 2) 单个任务缓存查询流程（CacheClient.queryWithMutex）

```mermaid
flowchart TD
    A[传入 taskId + InternshipTask.class] --> B[拼接缓存key]
    B --> C[读Redis String缓存]
    C --> D{命中?}
    D -- 命中空值__NULL__ --> E[返回null 防穿透]
    D -- 命中JSON --> F[ObjectMapper.readValue(json, InternshipTask.class)]
    F --> G[返回InternshipTask对象]
    D -- 未命中 --> H[尝试Redis互斥锁 setIfAbsent]
    H --> I{拿到锁?}
    I -- 否 --> J[短暂sleep后重试]
    J --> B
    I -- 是 --> K[二次检查缓存]
    K --> L{二次检查命中?}
    L -- 是 --> M[按命中分支返回]
    L -- 否 --> N[回源DB taskMapper.selectById]
    N --> O{DB有数据?}
    O -- 否 --> P[写__NULL__短TTL]
    P --> E
    O -- 是 --> Q[写缓存并加TTL随机抖动]
    Q --> G
    G --> R[finally释放锁]
    E --> R
```

## 3) 关键点说明

- `InternshipTask.class` 是类型元信息，用于 JSON -> 对象反序列化。
- 空值缓存用于防缓存穿透。
- 互斥锁用于防缓存击穿。
- TTL 随机抖动用于缓解缓存雪崩。
