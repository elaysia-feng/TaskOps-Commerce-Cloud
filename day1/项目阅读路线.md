# TaskOps Cloud 项目阅读路线（快速看懂版）

目标：先看懂“系统怎么跑”，再看懂“代码怎么串”，最后看懂“哪里能改”。

## 1. 先看什么（10 分钟）

1. `README.md`  
看整体架构、模块职责、启动方式。

2. `day1/项目流程图.md`  
先建立脑图：`Nginx -> Gateway -> auth/task/admin`，再去看代码会快很多。

3. `pom.xml`（根目录）  
看模块结构：
- `cloud-common`
- `auth-service`
- `task-service`
- `admin-service`
- `gateway-service`

## 2. 第二步看请求主链路（20 分钟）

先只追一条最关键链路：登录后创建任务。

1. 网关入口  
- `gateway-service/src/main/resources/application.yml`（路由）
- `gateway-service/src/main/java/com/elias/gateway/filter/JwtGlobalFilter.java`（鉴权与透传头）

2. 认证服务（拿 token）  
- `auth-service/src/main/java/com/elias/auth/controller/AuthController.java`
- `auth-service/src/main/java/com/elias/auth/service/impl/AuthAppServiceImpl.java`
- `auth-service/src/main/java/com/elias/auth/security/JwtTokenProvider.java`

3. 任务服务（业务核心）  
- `task-service/src/main/java/com/elias/task/controller/TaskController.java`
- `task-service/src/main/java/com/elias/task/service/impl/TaskAppServiceImpl.java`
- `task-service/src/main/java/com/elias/task/client/AuthClient.java`（Feign 调 auth）

结论你要能说出来：
1. token 在哪里签发
2. token 在哪里校验
3. 用户信息如何从网关传到下游服务
4. task-service 为什么要调 auth-service

## 3. 第三步看数据层（15 分钟）

1. 表结构  
- `sql/init.sql`

2. 实体与 Mapper  
- `auth-service/src/main/java/com/elias/auth/entity/*`
- `auth-service/src/main/java/com/elias/auth/mapper/*`
- `task-service/src/main/java/com/elias/task/entity/*`
- `task-service/src/main/java/com/elias/task/mapper/*`

重点：
1. `sys_user/sys_role/sys_user_role`（权限模型）
2. `internship_task`（核心业务表）
3. `login_log/operation_log`（观测与审计）

## 4. 第四步看配置中心与注册发现（10 分钟）

1. 每个服务的 `bootstrap.yml`  
看 Nacos 的 `server-addr`、`config import`。

2. 服务启动类  
看 `@EnableDiscoveryClient`、`@EnableFeignClients`、`@MapperScan` 等关键注解。

## 5. 第五步看公共层（5 分钟）

- `cloud-common/src/main/java/com/elias/common/ApiResponse.java`
- `cloud-common/src/main/java/com/elias/common/exception/*`
- `cloud-common/src/main/java/com/elias/common/context/UserContext.java`

作用：统一返回、异常处理、用户上下文透传。

## 6. 你要按这个顺序本地调通（建议）

1. 启动中间件（你已在 Ubuntu 跑）  
2. 本地启动 `auth-service`，先测登录注册  
3. 本地启动 `task-service`，测创建任务/搜索/热榜  
4. 本地启动 `admin-service`，测 dashboard  
5. 最后启动 `gateway-service`，统一走网关测一遍

## 7. 第一次阅读时不要陷进去的点

1. 不要先抠 SQL 细节，先看请求链路  
2. 不要先看所有类，先看 Controller + ServiceImpl  
3. 看到 Feign 调用，先确认接口路径是否对齐，再看实现

## 8. 看懂后的第一批可改进点（你后续可做）

1. 增加 task 更新/删除接口（现在以创建和查询为主）
2. 补用户密码加密策略与初始化数据一致性
3. 给 admin 看板增加分页和统计维度
4. 增加集成测试（网关鉴权 + Feign 调用链）
