# 项目问题汇总（本次对话）

## 1. 环境与部署问题

### 1.1 把中间件跑在 Ubuntu，微服务本地运行
- 结论：可行，且适合当前开发模式。
- 中间件最终保留：`Nacos + MySQL + Redis`。

### 1.2 docker-compose 文件误配
- 问题：一开始写了 ES/Kibana，但项目当前未使用。
- 处理：已从 `day2/docker-compose.yml` 移除 ES/Kibana，仅保留实际中间件。

### 1.3 Redis 端口冲突
- 现象：`6379 address already in use`。
- 原因：Ubuntu 宿主机已有 Redis 占用 6379。
- 处理：容器端口改为 `6380:6379`。

### 1.4 MySQL 容器启动后自动退出
- 现象：`taskops-mysql` 短时间 `Up` 后退出。
- 处理：
  1. 使用稳定镜像 `mysql:8.0.18`。
  2. 去掉不必要启动参数。
  3. 修正初始化/挂载配置。

### 1.5 Nacos 注册失败（Client not connected STARTING）
- 现象：服务启动时报 Nacos 注册失败。
- 原因：Nacos 2.x gRPC 端口未开放。
- 处理：在 compose 暴露 `9848`、`9849`（除 `8848` 外）。

### 1.6 MySQL 数据丢失风险
- 问题：容器重建后库表消失。
- 原因：MySQL 未做数据持久化。
- 处理：挂载 `/home/elias/springCloud/mysql-data:/var/lib/mysql`。

## 2. 网络与连接问题

### 2.1 DataGrip/IDEA 连接被拒绝
- 现象：`Connection refused`。
- 原因：对应容器未真正启动或端口未监听。
- 处理：先修复 compose 启动错误，再校验 `docker ps` 和监听端口。

### 2.2 MySQL 登录命令参数错误
- 现象：用 `-p 3307` 报认证错误。
- 原因：端口参数应为 `-P`（大写）。
- 正确命令：`mysql -h 127.0.0.1 -P 3307 -uroot -proot`。

### 2.3 Public Key Retrieval is not allowed
- 原因：MySQL8 非 SSL + 认证握手限制。
- 处理：JDBC URL 增加 `allowPublicKeyRetrieval=true`。

### 2.4 Unknown database 'internship_taskops'
- 原因：连接实例中不存在目标库（可能连错实例或重建后丢库）。
- 处理：在当前容器内创建库并执行初始化 SQL。

## 3. 代码依赖与启动问题

### 3.1 Feign 调用失败：No Feign Client for loadBalancing
- 原因：缺少 `spring-cloud-starter-loadbalancer`。
- 处理：为 `admin-service`、`task-service` 补充该依赖。

### 3.2 错误码定义方式问题
- 问题：散落魔法数字（如 `4010`、`4030`）不利于前后端对齐。
- 处理：在 `cloud-common` 新增统一 `ErrorCode`，并支持 `BizException(ErrorCode)`。

### 3.3 枚举定义报错
- 原因：新枚举常量写在分号 `;` 后面。
- 规则：所有枚举常量必须在分号之前。

## 4. 认证与业务逻辑问题

### 4.1 register 逻辑缺陷（初版）
- 问题：
  1. 密码明文存储。
  2. 每次注册都插入 `USER` 角色。
  3. 未插入 `sys_user_role` 关系。
- 处理：改为密码加密、查角色、插入用户-角色关联，并事务保护。

### 4.2 角色不存在的处理
- 结论：`request` 不应接收角色（防越权）；后端固定绑定默认 `USER`。
- 建议：默认角色不存在时应报错或初始化，不应静默跳过。

### 4.3 登录失败风控逻辑
- 已确认 Redis 用法：
  - `auth:fail:{username}` 失败计数（10分钟过期）
  - `auth:lock:{username}` 临时锁定（5分钟）
- 当前项目未把用户名/密码写入 Redis（这是正确做法）。

## 5. Redis 认知与使用问题

### 5.1 Redis 五种数据类型确认
- `String / Hash / List / Set / ZSet`。
- 当前项目实际使用：`String`（风控）+ `ZSet`（热榜）。

### 5.2 `:` 命名与“目录”误解
- 结论：`auth:fail:admin` 不是目录结构，只是 key 命名约定。
- 图形化工具显示为树是 UI 分组效果。

### 5.3 `Duration.ofMinutes(10)` 含义
- 结论：是 10 分钟，不是 10 秒（10 秒应是 `ofSeconds(10)`）。

## 6. Spring/Java 常见误区与修正

### 6.1 `@Transactional` 是否必须 `@Enable...`
- 结论：Spring Boot 常规依赖下默认可用，通常不需要额外显式开启。

### 6.2 `if (Boolean)` 与 `Boolean.TRUE.equals(...)`
- 问题：包装类型 `Boolean` 可能为 `null`，直接 `if(b)` 可能 NPE。
- 处理：用 `Boolean.TRUE.equals(b)` 做空安全判断。

### 6.3 `throw` 后还写 `return`
- 结论：`throw` 后方法已中断，后续 `return` 不可达且多余。

### 6.4 JSON 解析错误
- 原因：请求体 JSON 格式不合法（多逗号/引号错误）。

## 7. AOP 与上下文改造

### 7.1 已新增 AOP 能力
- `PerformanceLogAspect`：统一记录 controller/service 耗时与异常。
- `@TimeLimit + TimeLimitAspect`：可设置方法耗时阈值，支持超时仅告警或抛异常。

### 7.2 重复时间字段赋值问题
- 讨论结论：`createdAt/updatedAt` 更推荐用 MyBatis-Plus 自动填充，而非 AOP。

### 7.3 已接入 ThreadLocal 用户上下文
- 新增 `UserContextFilter`：
  1. 每次请求从头读取 `X-User-Id/X-Username/X-Roles`。
  2. 写入 `UserContext(ThreadLocal)`。
  3. `finally` 清理避免线程复用污染。
- 控制器已可直接使用：`UserContext.userId()` / `UserContext.roles()`。
